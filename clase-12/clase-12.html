Clase 12 - Operadores

Operador Ternario

Se utiliza una condición, la condición resuelve true o false. En el caso uno, se escribe la instrucción a ejecutar si es verdadero, y en el caso 2, si es falsa.

  • El operador ternario ya tiene un return implícito, es decir, que me resulta muy útil retornar un dato
  • Un caso muy utilizado es para levantar un array del local storage

Operador Lógico && AND

Si la condición es verdadera, devuelve el valor de la segunda parte, es una simplificación del condicional if, en una sola línea de código.

  • También me sirve para retornar valores, ejemplo: let vacio = carrito.length === 0 && "Esto está vacío"; Lo que se dice es que si el carrito no contiene valores, entonces, almacene el string "Esto está vacío" en la variable vacio

Estructura

Lo que dice es: Si la variable nombreCliente es estrictamente igual a "Samuel", entonces, ejecuta el alert. Sino, el código no se ejecuta.

nombreCliente === "Samuel" && alert("Bienvenido Samuel");

Operador Lógico || OR

Ahora trabajamos con dos operandos y ya no tenemos una condición explícita. Entonces devuelve el primer valor que sea verdadero

  • Lo podemos utilizar para pedir datos al usuario, por ejemplo:
    let nombre = prompt("Ingrese su nombre:");
    console.log(nombre || "El usuario no ingresó su nombre");
Tenemos que saber que en JS se denomina valores "falsy" a los siguientes: false, 0, "", null, undefined, NaN

Operador Nullish ?? Coalescing

Es una simplificación del operador OR, trabaja igual que el OR, pero sólo admite como valores "nullish" a null y el undefined

Estructura

console.log(0 ?? "Verdadero");

Acceso Condicional a un objeto

Cuando queremos acceder a un objeto que no existe, naturalmente nos retorna un error. Pero si usamos el operador ? sobre la referencia, podemos condicionar su acceso y tener un mejor control del error.

Desestructuración de objetos

Es una forma de extraer propiedades de un objeto y asignarlas a variables. Se trabaja una copia del objeto, y convierte cada uno de los datos dentro del objeto a una variable independiente.

  • Puedo declarar con const o let. Adentor de las llaves puedo poner el nombre de las propiedades que quiero extraer y en memoria se crean esas variables a las cuales cuedo acceder de forma independiente después.
  • Si llamo a una propiedad no definida, retorna undefined.
  • Si quiero cambiar el nombre de la variable que se crea, puedo hacerlo de la siguiente forma:
    let {nombre, apellido, edad, curso:carrera} = alumno;
    console.log(carrera);

Estructura

let {nombre, apellido, edad, curso} = alumno;
console.log(nombre);

Desestructuración por parámetros

Es una forma de desestructurar un objeto que se pasa como parámetro a una función

Estructura

const saludar = ({nombre, apellido, edad}) => {
console.log(`Hola ${nombre}${apellido} tienes ${edad} años`);
}
saludar(alumno);

Desestructuración de array

Esto es una forma de extraer elementos de un array, trabaja por orden, es decir, el primer valor del array se ubicará en la primera variable que coloquemos, y así sucesivamente.

  • Si quieres saltar un valor, es decir no pasarlo a variable, entonces, se deja el espacio libre: let [uno, , tres, , cinco] = array;

Estructura

let arrayNumeros = [1, 2, 3,4, 5];
let [primero, segundo, tercero, cuarto, quinto] = arrayNumeros;
